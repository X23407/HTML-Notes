Regular Expressions (Regex) Tutorial

A regular expression (regex) is a sequence of characters that defines a search pattern. Regex is widely used for searching, matching, and replacing text in programming languages, text editors, and command-line tools.

Basic Syntax

  • Literal characters: Match themselves (e.g., cat matches "cat").
  • Metacharacters: Special symbols like ., *, +, ?.
  • Character classes: [abc] matches "a", "b", or "c".
  • Ranges: [0-9] matches any digit.
  • Anchors: ^ (start of string), $ (end of string).

Example: ^Hello matches any string starting with "Hello".

Quantifiers

  • * → 0 or more occurrences
  • + → 1 or more occurrences
  • ? → 0 or 1 occurrence
  • {n} → exactly n occurrences
  • {n,} → n or more occurrences
  • {n,m} → between n and m occurrences

Example: a{2,4} matches "aa", "aaa", or "aaaa".

Groups and Alternation

  • Grouping: (abc) treats "abc" as a single unit.
  • Alternation: a|b matches "a" or "b".
  • Capturing groups: allow extracting matched substrings.
  • Non-capturing groups: (?:abc) groups without capturing.

Example: (dog|cat) matches "dog" or "cat".

Advanced Features

  • Lookahead: X(?=Y) matches X only if followed by Y.
  • Negative Lookahead: X(?!Y) matches X only if not followed by Y.
  • Lookbehind: (?<=Y)X matches X only if preceded by Y.
  • Flags: /i (case-insensitive), /g (global), /m (multiline).

Example: \d+(?=px) matches numbers followed by "px".

Practical Examples

  • Email validation: ^[\w.-]+@[\w.-]+\.[a-zA-Z]{2,}$
  • Phone number: ^\d{10}$ (matches 10 digits)
  • IP address: ^(\d{1,3}\.){3}\d{1,3}$

Conclusion

Regex is a powerful tool for text processing. By combining metacharacters, quantifiers, groups, and assertions, you can build complex patterns to match almost any text structure. Practice with small examples and gradually move to advanced use cases.
